% ABSTRACT
% ============================================================================
\begin{abstract}
AI-assisted software development has transformed how developers work, yet context loss during long-term projects remains a critical challenge. When development sessions span weeks or months, interruptions and context window limitations cause developers to spend 15--25 minutes recovering context after each break. We present the Universal Workflow System (UWS), a novel git-native workflow management system that provides persistent context across sessions through checkpoint-based state recovery. UWS introduces a multi-agent architecture with seven specialized roles, a modular skill library, and phase-based workflow management---all built on version-controlled state files.

We evaluate UWS through comprehensive automated benchmarks comparing context recovery performance against baseline approaches including manual recovery and git-only workflows. Results show UWS achieves average context recovery in 42 milliseconds (vs.\ 5--10 minutes for baselines), representing a $>$99\% improvement. Reliability testing demonstrates 100\% checkpoint recovery success under failure injection, and ablation studies confirm the contribution of each architectural component. Case studies across ML research, LLM development, and software engineering domains demonstrate generalizability.

Our contributions include: (1) a novel git-native state persistence mechanism for workflow systems, (2) a complete open-source implementation with comprehensive test suite, (3) empirical evaluation demonstrating significant performance improvements, and (4) a replication package enabling reproducibility.
\end{abstract}
